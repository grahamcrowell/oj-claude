# oj-claude v0.1.0 Test Plan

Validates the changes introduced by the `Release v0.1.0` commit (`f91e124` on
branch `oj-0.1.0-improvements`) against the prior release `v0.0.11` (`e04b624`),
AND confirms that the prioritized recommendations from the external plugin
assessment (section 4, "Improvement opportunities (prioritized)") were followed
in this release or are documented as deferred.

## Scope

v0.1.0 is a structural / hygiene release. Every change is one of:
- a file relocation (agents/ -> reference/),
- a shell-behavior change in `bin/oj-helper inject-profile`,
- a frontmatter edit (agent profiles + skill headers),
- a content split of `CONDUCTOR.md`.

No stakeholder-agent logic, model roster, or backlog protocol changed. The test
plan therefore weights toward (a) mechanical / automated assertions, (b) a small
set of runtime behavioral checks that need a live session, and (c) a
traceability pass that maps each assessment recommendation to its v0.1.0
disposition.

## Test tiers

- **Tier A - Automated regression** (existing harnesses). Must be green before
  any manual test is meaningful. Gates the release.
- **Tier B - Mechanical assertions** (new, change-specific). Grep/diff/awk
  checks proving each relocation and frontmatter edit landed exactly.
- **Tier C - Runtime behavioral** (live session). Proves the injected context
  and agent registry actually reflect the changes.
- **Tier D - Assessment recommendation traceability**. Confirms each of the 14
  prioritized recommendations is implemented (and tested), or is a documented
  deferral (asserted negatively so we do not falsely claim it).

## Tier C - Runtime behavioral (live Claude Code session)

Install the plugin at `f91e124` and start a real session. These prove the file
changes produce the intended runtime effect.

### T-R1 - Agent registry is clean (the C5 headline fix)

- **Before/after anchor**: under v0.0.11 the plugin registers bogus agent types
  `oj:_preamble`, `oj:index`, and 16 `oj:senior-<role>-compact`, each with the
  useless description "Agent from oj plugin", alongside the 16 real

- **Test**: in a v0.1.0 session, enumerate available `oj:*` agent types.
- **Pass**: exactly 16 `oj:senior-*` types; NO `oj:_preamble`, NO `oj:index`,
  NO `oj:senior-*-compact`. Each of the 16 shows its new routable `description`
  (C4), not "Agent from oj plugin".

### T-R2 - Profile injection at spawn

- **Test**: trigger `inject-profile` for a named expert (spawn a
  `general-purpose` subagent, or drive `/oj:run-task` on a Moderate item).
- **Pass**: the subagent receives preamble + full profile PROSE, with NO YAML
  frontmatter leaking (runtime confirmation of T-B5.4).

### T-R3 - Slim injection at SessionStart (C1)

- **Test**: start a fresh session; inspect the SessionStart-injected CONDUCTOR
  context.
- **Pass**: injected core has role, absolute constraints, two-dimensional
  triage, stakeholder summary, and the load pointer to
  `reference/execution-protocol.md`; it does NOT contain the full execution

### T-R4 - Execution-protocol loads on demand (C1)

## Exit criteria

Release is validated for v0.1.0 when:

1. **Tier A**: T-A1..T-A5 all exit 0 (validate-plugin 8/8, plugin-e2e 25/25).
2. **Tier B**: T-B1.* .. T-B7.* all pass. Any failure is a hard block - these
   are the mechanical guarantees of the release.
3. **Tier C**: T-R1..T-R7 pass, OR are explicitly recorded as BLOCKED (no
   `claude` CLI) with Tier B covering the static half. T-R1, T-R3, T-R4 are the
   highest-value runtime checks; do not ship without them if a live session is
   available.
4. **Tier D**: every DONE recommendation passes its test; every DEFERRED
   recommendation passes its negative assertion (absent AND documented);
   PARTIAL items (4, 13) match their stated disposition exactly. Recommendation
   12 was completed via the `senior-sre.md` follow-up, so T-D12a/T-D12b both
   pass on the current tree; there are no remaining EXPECTED FAILs.

## Regression guards to keep

Fold Tier B and the Tier D assertions into a committed
`scripts/tests/v0.1.0-regression-test.sh` wired into pre-commit / CI so these
cannot silently regress:
- `agents/compact` string never returns to `bin/oj-helper` (T-B5.5).
- `agents/` stays at exactly 16 files, no `_`-prefixed or nested entries (T-B5.2).
- injected CONDUCTOR byte budget stays under the threshold (T-B1.3).
- the six side-effecting skills keep `disable-model-invocation: true` (T-B6.1).
- CONDUCTOR core stays free of model names (T-D13 core assertion).
- expert profiles gain no `tools`/`model`/`effort` keys until the native
  migration lands as its own release (T-D4 negative).

## Notes / risks

- The ~13.3KB injection figure is from the CHANGELOG; assert against a
  threshold, not equality - minor edits shift it.
- T-B5.4's frontmatter-strip awk only removes a block beginning on line 1. A
  future profile that does not lead with `---` is injected whole (deliberate,
  tested). Keep the negative assertion so a rewrite of the stripper does not
  start eating body `---` rules.
- Tier C/D-runtime observations depend on the host harness exposing the agent
  registry and injected context. If hidden, capture via `OJ_HOOK_DEBUG=1`
  `oj-helper` debug output as the fallback source of truth.
