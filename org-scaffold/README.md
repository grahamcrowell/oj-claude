## What this repo is

This is the org-level coordination hub for [ORG_NAME]'s OpenJunto Claude Code
configuration. It holds shared reference material, an org-level backlog, a
cross-project context map (`llms.txt`), and slash commands that span the
individual project repos this org manages.

It is **not** a per-developer Claude Code installation — those live in each
developer's `~/.claude/` directory and are managed by the OpenJunto installer.
It is also **not** a per-project `.claude/` directory — those live alongside
the code in each project repo.

## How it relates to per-project `.claude/` directories

OpenJunto uses a three-tier inheritance model:

1. **Developer-global** (`~/.claude/`) — installed by `make install`. Contains
   the manager protocol, expert profiles, baseline reference files, and
   universal slash commands. One per developer machine.
2. **Org coordination repo** (this repo) — shared across the team. Contains
   org-validated facts, the org backlog, the cross-project context map, and
   org-scope slash commands. One per organization.
3. **Per-project `.claude/`** — lives in each project repo. Contains project
   backlog, session state, project-scope reference, and project artifacts.
   One per project repo.

Claude Code reads tiers 1 and 3 automatically. **Tier 2 (this repo) is a
source of shared reference, not a config directory that Claude Code reads
automatically.** Developers consume org content explicitly — by pointing
Claude Code at this directory when running org-scope commands, by reading
specific files with the Read tool, or by copying selected reference files
into their `~/.claude/reference/` for routine availability.

The org/project boundary is defined in `D80-org-coordination.md` § Reference
File Scoping. See also `reference/org-coordination-guide.md` for day-to-day
operational guidance.

## How to use it

**Find the org backlog**: `BACKLOG.md` at the repo root holds org-level
coordination items. Per-project items live in each project's `.claude/BACKLOG.md`.

**Discover the projects**: `llms.txt` lists the repos this org coordinates
and their `.claude/` entry points. Keep it current as repos join or leave
the org.

**Run an org-scope command**: Point Claude Code at this directory and invoke
the command. For example, to run the org health check:

```
cd [ORG_REPO_PATH]
claude
> /health-check
```

To triage and route a new inbound request:

```
cd [ORG_REPO_PATH]
claude
> /intake
```

**Reference org-validated facts from a project**: From within a project
working directory, read the org reference file directly:

```
> Read [ORG_REPO_PATH]/reference/validated-facts-org.md
```

Or copy the file into the developer's `~/.claude/reference/` for routine
availability.
