# /health-check — Org Coordination Health Check

**Constraint**: This command is **read-only**. Report observations; do not
modify any file. Auto-fix is explicitly out of scope — the operator carries
findings to the right repo or backlog item.

## Purpose

Verify the org `.claude` coordination repo and the projects it coordinates
are in coherent state. Surface drift (stale facts, missing entries,
committed volatile state) before it causes cross-project confusion.

## Execution

Run each check in order. For each, emit one line in the structured output
format defined below. Continue on failure — collect all findings before
emitting the final report.

### Check 1 — Org repo state

- Confirm `llms.txt` exists at the repo root and is non-empty.
- Confirm `BACKLOG.md` exists at the repo root.
- Confirm `.gitignore` exists and contains a rule excluding `state/*`
  (e.g., a literal `state/*` line, or equivalent pattern).
- Emit:
  - `[PASS] llms.txt present (N projects listed)` — N = count of non-comment, non-blank lines under `## Projects` not equal to placeholder text.
  - `[PASS] BACKLOG.md present (N open items)` — N = count of `### BACK-` headings under `## Active`.
  - `[PASS] state/ gitignored` — if `.gitignore` excludes `state/*`.
  - `[FAIL]` variants if any of the above is missing or misconfigured.

### Check 2 — Project inventory

For each project listed in `llms.txt` under `## Projects`:

- Parse the entry as `[REPO_NAME]: [path or URL] — [description]`.
- If the value looks like a local filesystem path (starts with `/`, `~`, or
  `.`), check whether that path exists and contains a `.claude/` directory.
- If the value looks like a URL (starts with `http://`, `https://`, `git@`),
  do not attempt to fetch it — emit an advisory line noting the remote and
  that reachability was not verified.
- Skip placeholder entries (any value containing `[REPO_NAME` or the literal
  bracketed placeholders); emit `[WARN]` if placeholders remain after the
  first project has been added.
- Emit:
  - `[PASS] [REPO_NAME]: local path exists and has .claude/`
  - `[WARN] [REPO_NAME]: remote URL — reachability not verified`
  - `[FAIL] [REPO_NAME]: local path missing or has no .claude/`
  - `[WARN] llms.txt still contains placeholder entries` (once, if any)

### Check 3 — Stale state detection

- Check whether `state/session.md` (or any file directly under `state/`
  other than `.gitkeep`) is tracked by git.
- If git is unavailable in the working environment, emit a single advisory
  noting the check was skipped.
- Emit:
  - `[PASS] state/ contains no tracked files (besides .gitkeep)`
  - `[FAIL] state/session.md is tracked by git — add to .gitignore`
  - `[WARN] git unavailable — stale-state check skipped`

### Check 4 — Org backlog coherence

Scan `BACKLOG.md`:

- An item is **In Progress** if its `**Status**:` line contains `In Progress`.
- An item has **no owner** if it lacks an `**Owner**:` line, or its owner
  line value is empty or contains `unassigned`.
- Flag items marked **In Progress** with no owner.
- Flag items in `## Active` whose description text references a BACK-NNN
  identifier under `## Completed` as a blocker (simple substring match
  against completed-section headings is sufficient).
- Emit:
  - `[PASS] no in-progress items without owners`
  - `[WARN] BACK-NNN marked In Progress but has no owner` (one line per item)
  - `[WARN] BACK-NNN appears blocked by completed item BACK-MMM`

### Check 5 — Reference freshness

- Report the age (in days) of the last modification to
  `reference/validated-facts-org.md`.
- Emit:
  - `[PASS] validated-facts-org.md last updated N days ago` — if N ≤ 30.
  - `[WARN] validated-facts-org.md last updated N days ago — consider reviewing` — if N > 30.
  - `[FAIL] reference/validated-facts-org.md missing` — if the file is absent.

## Output format

Print one line per finding, prefixed with `[PASS]`, `[WARN]`, or `[FAIL]`.
After all checks, print a summary footer:

```
[PASS] llms.txt present (3 projects listed)
[PASS] BACKLOG.md present (2 open items)
[PASS] state/ gitignored
[PASS] [REPO_NAME_1]: local path exists and has .claude/
[WARN] [REPO_NAME_2]: remote URL — reachability not verified
[PASS] state/ contains no tracked files (besides .gitkeep)
[PASS] no in-progress items without owners
[WARN] validated-facts-org.md last updated 47 days ago — consider reviewing

---
Summary: 5 PASS, 2 WARN, 0 FAIL
```

## Failure handling

- Any `[FAIL]` line should be followed by a one-sentence remediation hint
  (e.g., `→ add 'state/*' to .gitignore`).
- Do not exit non-zero or block further work. The operator decides
  follow-up.
- Do not modify any file as part of the check, even to fix obvious errors.
